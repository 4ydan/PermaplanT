# Use Case: Winds Layer

## Summary

- **Scope:** Wind Layer
- **Level:** User Goal
- **Actors:** App User
- **Brief:** The user can add, edit, move and delete wind indicators in their plan using the wind layer
- **Status:** Draft

## Scenarios

- **Precondition:**
  The user has opened the app and has selected the winds layer.
- **Main success scenario:**
  The user successfully adds, edits, moves and deletes wind indicators in their landscape plan using the wind layer. This includes positioning the indicators in areas of the landscape that are more windy and adjusting the strength of the wind indicator.
- **Alternative scenario:**
  The user accidentally adds or moves a wind indicator in the wrong location and uses the app's undo function to correct the mistake.
- **Error scenario:**
  The user attempts to add or edit a wind indicator but the app is experiencing technical difficulties and is unable to complete the request, displaying an error message.
- **Postcondition:**
  The user's landscape plan includes the added, edited, moved or deleted wind indicators as desired.
